# Product Overview

## Selected Framing

- 프로젝트 이름: `card-mizer`
- 한 줄 소개: 카드 실적과 혜택 우선순위를 계산해 결제 카드를 추천하는 Kotlin 백엔드
- 핵심 문제: 여러 카드의 월간 실적 구간과 혜택 우선순위를 함께 고려한 결제 판단 비용이 크다

## Portfolio Positioning

- GitHub 프로필에서 가장 먼저 보여줄 대표 Kotlin 프로젝트로 둔다.
- "실사용 문제를 도메인 로직으로 푼 Spring 백엔드"라는 메시지를 전면에 둔다.
- 단순 CRUD보다 아키텍처 결정, 도메인 모델링, 문서화 습관을 증명하는 저장소로 설계한다.
- 이후 블로그에서는 헥사고날 구조, 추천 규칙 모델링, 멀티모듈 설계 결정을 각각 독립 포스트로 확장한다.

## Current Repository Slice

- 현재 저장소는 JPA/H2 기본 경로와 시드 데이터로 동작하는 백엔드 데모를 포함한다.
- 추천 API, 실적 조회 API, 사용 내역 입력 API가 구현돼 있다.
- Spring Boot 정적 리소스로 제공되는 데모 화면이 포함돼 있지만, 별도 프론트엔드 제품을 목표로 하지는 않는다.
- 카드 등록, 우선순위 조정, 카드 정책 전체 교체/부분 수정까지 구현돼 있다.

## Target MVP Scope

포함:

- 카드 등록
- 카드별 실적 정책 등록
- 월별 사용 내역 수동 입력
- 현재 실적 현황 조회
- 결제 금액 기준 추천 카드 계산
- 카드 우선순위 조정

제외:

- 실제 카드사 API 연동
- 푸시/메신저 알림
- 별도 프론트엔드 애플리케이션
- 복수 사용자 지원
- 고급 혜택 시뮬레이션

## Success Criteria

- 실적 조회와 추천 유스케이스가 core 중심으로 설명 가능하다.
- 멀티모듈 경계와 포트/어댑터 경계가 문서와 코드에서 일치한다.
- README 상단만 읽어도 문제, 해결 방식, 현재 구현 상태가 한 번에 보인다.
