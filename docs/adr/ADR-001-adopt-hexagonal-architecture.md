# ADR-001: Adopt Hexagonal Architecture

## Status

Accepted

## Context

이 프로젝트는 카드별 실적 정책과 결제 추천 규칙처럼 도메인 판단이 중심이 된다. 향후 외부 카드사 연동, 알림, 스케줄러가 추가될 가능성도 있다.

## Decision

도메인 코어를 포트 기반으로 정의하고, API/저장소/외부 연동은 어댑터로 분리한다.

## Consequences

- 도메인 로직 테스트가 쉬워진다.
- 저장소나 외부 연동 변경이 코어에 덜 영향을 준다.
- 초기 구조 설계 비용은 늘어나지만, 포트폴리오와 장기 유지보수 관점에서는 이점이 크다.
