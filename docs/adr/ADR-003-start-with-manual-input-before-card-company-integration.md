# ADR-003: Start with Manual Input Before Card Company Integration

## Status

Accepted

## Context

카드 추천 엔진이 동작하려면 사용 내역 데이터가 필요하다. 데이터를 확보하는 경로는 크게 세 가지다.

1. 카드사 API 또는 마이데이터 연동으로 자동 수집
2. CSV/엑셀 일괄 업로드
3. REST API를 통한 수동 입력

카드사 API 연동은 OAuth 인증, 인증서 관리, API 장애 처리, 카드사별 데이터 포맷 차이 등 외부 의존성이 크다. 포트폴리오 단계에서 이 복잡성을 감수하면 핵심인 추천 로직 검증이 늦어진다.

## Decision

첫 번째 데이터 입력 경로는 REST API를 통한 수동 입력으로 한다. `SpendingRecord`는 입력 출처에 의존하지 않는 범용 도메인 모델로 설계하고, 카드사 연동은 이후 어댑터로 추가한다.

## Consequences

- 외부 API 계약 없이 추천 엔진을 바로 검증할 수 있다.
- `SpendingRecord`가 입력 출처에 무관하게 동일한 구조를 유지하므로, 이후 카드사 API 어댑터를 붙여도 도메인 로직은 변경되지 않는다.
- 사용자가 직접 입력해야 하므로 실사용 편의성은 떨어진다. 이 부분은 데모 시나리오 시드 데이터로 보완한다.
- 카드사 API에서 제공하는 거래 고유번호, 승인 코드, MCC 코드 같은 필드는 현재 모델에 없다. 연동 시점에 필드를 확장하거나 별도 raw 레코드를 두는 판단이 필요하다.
- 정규화 로직이 수동 입력 기준으로 먼저 안정화되므로, 이후 카드사 데이터가 들어올 때 정규화 규칙을 소스별로 분기하는 구조가 자연스럽게 나온다.
